Configuring Chroma Manager

  

Before completing this procedure, refer to the documentation provided by your storage solution provider for configuration steps to be completed outside the Chroma Manager user interface. A superuser is created as part of this initial configuration.

A final step to configuring Chroma Manager is to set up user accounts. User accounts are set up on the Configuration > Users page by clicking +Create User.

When a user account is created, the user is placed into one of three roles:

•Superuser  - A superuser can create, modify (change passwords), and delete users by going to Configuration > Users (this tab is only visible to superusers).  A superuser cannot delete their own account, but a superuser can create or delete another superuser.

•File system administrator - A file system administrator can perform all the actions of a superuser except creating or modifying another user.

•File system user - A file system user can only view content in Chroma Manager. Users log in by clicking Login in the upper right corner of the screen and log out by clicking Logout.

A superuser or file system administrator must be logged in to perform any actions that modify the system, such as starting a file system or adding a server.

Users can modify their own account when they are logged in by clicking Account near the upper right corner of the screen. These options can be set by the user:

•Details - Username, email address, and first and last name can be changed.

•Password - Password can be changed and confirmed.

•Email Notifications - The types of events for which this account is to receive emailed notifications can be selected from a checklist. If no notifications are selected, email notifications will be sent for all alerts except “Host contact alerts”.

Note: Notification of alerts by email is not enabled by default. See the documentation provided by your storage solution provider for how to configure Chroma Manager to enable emailing of alert notifications.

Note: Unauthenticated users can access the static HTML content on the Chroma Manager server, but the display will not be populated with current information about the system unless the user is authenticated. See the documentation provided by your storage solution provider for how to set a Chroma Manager configuration setting to require all users to log in to see any data.
