Monitoring a Lustre File System

  

A file system is monitored in Chroma Manager using the Dashboard, Alerts, Events, and Logs pages.

Dashboard

The Dashboard displays a set of graphs that provide usage and performance data at several levels in the file systems being monitored.  At the top level, an aggregate view of all file systems monitored by Chroma Manager is displayed. Views of individual file systems, servers, and devices are displayed using the selection fields at the top of the page.

The following graphs appear in one or more views depending on the content of the selection fields at the top of the Dashboard page:

•Percent Capacity Used - The left bar indicates the percentage of file system disk space currently being used. The right bar shows the percentage of available files (inodes) that have been used. Depending on the view selected, data is displayed for all file systems, all servers, or a single file system.

•Clients Connected - Shows the number of clients that have mounted the file system over time. Depending on the view selected, data is displayed for all file systems, all servers, or a single file system.

•Metadata Operations - Shows the number of metadata I/O operations over time. Depending on the view selected, data is displayed for all file systems, all servers, a single file system, or a single metadata server.

•Read/Write Bandwidth - Shows reads and write activity on targets over time. Depending on the view selected, data is displayed for all file systems, all servers, a single file system, or a single MDT or OST.

•Metadata Server - Shows the percentage of CPU and RAM resources used on the metadata server over time.  Depending on the view selected, data is displayed for metadata servers in all file systems being managed (view of all file systems or all servers displayed) or the metadata server in a single file system.

•Object Storage Servers - Shows the percentage of CPU and RAM resources used on object storage servers over time. Depending on the view selected, data is displayed for all file systems, all servers, or a single file system.

•CPU Usage - Shows the percent of CPU usage attributed to user-level processes, to system-level processes, and to processes in an IO Wait state over time. Data is displayed for the specific metadata or object storage server selected.

•Memory Usage - Shows the total memory available, the amount of memory used, the total swap space available, and the amount of swap space being used over time. Data is displayed for the specific metadata server or object storage server selected.

•Space Usage - Shows the percentage of file system space used on a target over time. Data is displayed for the specific metadata target or object storage target selected.

•File Usage - Shows the percentage of available files (inodes) used over time. Data is displayed for the specific metadata target selected.

•Read vs. Write - Shows the number of read and write operations for the object storage target over time. Data is displayed for the specific object storage target selected.

•Object Usage - Shows the percentage of metadata objects used over time. Data is displayed for the specific object storage target selected.

Data used to produce the graphics is saved for long-term use. Data is averaged and compressed over time so that the most recent data is stored and viewed at maximum resolution while aging data is stored and viewed at progressively lower resolutions over time.

 

Alerts, History, and Logs

Alerts – Shows active alerts and alert history.

History – Displays a history of events and allows filtering of events by host, severity, and event type.

Logs –  Displays log information and allows filtering by host, date range, and messages from Lustre or all sources.

 

For example, if a failover event takes place, the following occurs:

• An alert is displayed with a message that a server has failed over in the Notifications panel and on the Alerts page.

• A warning is recorded on the History page for each target that has failed over.

•The alert icon appears on the Configuration page for the file system. The server on which the target is now running is shown in the Started On column for that target.

•An email alert is sent to the system administrator. See the documentation provided by your storage solution provider for how to configure your mail server to enable and set up email alerts.
