Dashboard window


Contents

The Dashboard displays a set of charts that provide usage and performance data at several levels for each file system. At the top level, this page displays an aggregate view of all file systems you're currently monitoring. You can select to  monitor individual file systems, servers, or devices, using the drop-down menus at the top of the page. For example, to view charts for an individual server, select Servers in the left drop-down menu and then select the specific server.

At the top, the Dashboard lists the file system(s) being managed or monitored-only.  The following information is provided for each file system:

●File System name: The name assigned to this file system during its creation on the Configuration page.

●Type: Monitored or Managed.  Managed file systems are configured and managed for high availability (HA).  Managed file systems are both monitored and managed, whereas monitored file systems are monitored-only and do not support failover via Intel® Manager for Lustre* software.

●Space Used / Total: This indicates the amount of file system capacity consumed, versus the total file system capacity.

●Files Used / Total: This indicates the total number of inodes consumed by file creation versus the total number of inodes established for this file system.

●Clients: Indicates the number if clients accessing the file system at this moment.

Charts

The charts discussed next are displayed on the Dashboard. The following controls are present:

Change Duration: Five of the charts display information over time. Change Duration lets you change the time period for the range of data displayed on the chart. Each time-driven chart begins at a start time set in Duration and ends now.  You can set this to select Minutes, Hours, Days or Weeks, up to four weeks back in time and ending now. Each chart is updated continuously. For time-based charts, the most recent data displayed on the right.

Full Screen: Click Full Screen to fill the browser window with this chart. Click Exit Full Screen to return to the normal view.

Drag: This control lets you click and drag a chart, to change the chart ordering on the Dashboard.

The following charts are displayed on the Dashboard:

●Read/Write Heat Map - This chart shows the level of read/write activity for each OST in all file systems.  OSTs are displayed as rows, and consecutive time intervals are displayed as columns. The chart updates from right, to left. You can the monitor the level of read or write activity for given OST over time by looking across the chart. Read or write activity is displayed in increasingly darker shades, from light pink to red. Values range from 0% of maximum (0 megabytes per second and light pink), to 25%, 50%, 75% and 100% of maximum (and the darkest shade of red). The maximum and minimum displayed values are those measured at any given moment. Units vary based on the appropriate range of values. You can mouse over any cell on the heat map to identify a specific OST and its file system, its read or write activity, and the actual time of that activity measurement. You also set the time duration for the heat map, to hours, days, or weeks, and your selected time duration is then divided equally into the heat map’s twenty columns.  

●OST Balance - This chart shows the percentage of capacity currently being used for each OST. Each bar represents one OST and the chart ranges from 0 to 100% used, with the red representing the used capacity. Mouse over an individual OST to learn the number of bytes used versus the number of bytes free, and the total capacity of that OST. Depending on the view selected, data is displayed for all file systems, all servers, or a single file system. Note: If larger numbers of OSTs are present, this chart is replaced with the Percent Capacity Used chart.

●Percent Capacity Used - This chart shows the used-versus-free portion of the overall file system. The left bar indicates the percentage of file system disk space in use. The right bar shows the percentage of available files (inodes) that have been used. Depending on the view selected, data is displayed for all file systems, all servers, or a single file system. Note: If lesser numbers of OSTs are present, this chart is replaced with the OST Balance chart.

●Metadata Operations - This chart shows the number of metadata I/O operations over time. Depending on the view selected, data is displayed for all file systems, all servers, a single file system, or a single metadata server.

●Read/Write Bandwidth - Shows read and write activity on targets over time. Depending on the view selected, data is displayed for all file systems, a single file system, servers with at least one OST, or a single OST.

●Metadata Servers - This chart shows the percentage of CPU and RAM resources used on the metadata server over time. Depending on the view selected, data is displayed for metadata servers in all file systems being managed (view of all file systems or all servers displayed) or the metadata server in a single file system.

●Object Storage Servers - This chart shows the percentage of CPU and RAM resources used on object storage servers over time. Depending on the view selected, data is displayed for all file systems, all servers, or a single file system.

●CPU Usage - This chart shows the percent of CPU usage attributed to user-level processes, to system-level processes, and to processes in an IO Wait state over time. Data is displayed for the specific metadata or object storage server selected.

●Memory Usage - This chart shows the total memory available, the amount of memory used, the total swap space available, and the amount of swap space being used over time. Data is displayed for the specific metadata server or object storage server selected.

●Space Usage - This chart shows the percentage of file system space used on a target over time. Data is displayed for the specific metadata target or object storage target selected.

●File Usage - This chart shows the percentage of available files (inodes) used over time. Data is displayed for the specific metadata target selected.

●Read vs. Write - This chart shows the number of read and write operations for the object storage target over time. Data is displayed for the specific object storage target selected.

●Object Usage - This chart shows the percentage of metadata objects used over time. Data is displayed for the specific object storage target selected.

Data used to produce the charts is saved for long-term use. Data is averaged and compressed over time so that the most recent data is stored and viewed at maximum resolution while aging data is stored and viewed at progressively lower resolutions over time.
