Managing Chroma Storage

  

The following are common management tasks that can be completed in Chroma Manager.

Adding storage targets to a file system

To increase the size of a file system by adding an object storage target, complete these steps:

1.Go to Configuration > File Systems.

2.In the file system table displayed, click on the file system name to display the file system page.

3.Under Object Storage Targets, click + Create new OST.

4.Select the OST or OSTs to be added and click OK. The new OSTs will be displayed in the table of OSTs on the file system page.
Adding or deleting users

Go to Configuration > Users to create or delete a user account.
Reconfiguring failover servers

To modify the failover settings for volumes that are configured for high availability (status is configured-ha), complete these steps:

1.Go to Configuration > Volumes.

2.Adjust the options in the Primary Server and Failover Server columns as appropriate.
Starting or stopping a file system

To start a file system, complete these steps:

1.Go to Configuration > File Systems.

2.In the table entry for the file system, click Start. The metadata and object store targets will be started enabling the file system to be mounted by clients.

To stop or remove a file system from Chroma Manager, complete these steps:

1.Go to Configuration > File Systems.

2.In the file system table displayed, click Stop to stop the metadata and object store targets, making the file system unavailable to clients.

3.Click Remove to remove the file system from Chroma. File system contents will remain intact until volumes are re-used in another file system.
Starting or stopping a server for a management, metadata, or object storage target

To start or stop a server for a target, complete these steps:

1.Go to Configuration > File Systems and click the <name> of the file system in which the target is located. The file system page is displayed.

2.Locate the target name in the first column and click Start or Stop for that target.

•When an MGT is stopped, clients are unable to make new connections to file systems using the MGT, but the MDT and OSTs stay up if they are currently started.

•When an MDT is stopped, the file system becomes inoperable until the MDT is started again.

•When an OST is stopped, clients are unable to access the files stored on this OST.
Decommissioning a server for a management, metadata, or object storage target

To decommission a server for a target, complete these steps:

1.Go to Configuration > File Systems and click the <name> of the file system in which the target is located. The file system page is displayed.

2.Locate the target and in the Primary Server column for that target, click on the name of the server.

3.In the dialog that is displayed, click Remove to remove the server from the file system.

•When the server for an MGT is removed from Chroma Manager, the contents will remain intact until the volume is reused for a  new target.

•When the server for an MDT is removed from Chroma, any file systems or targets that rely on this server will also be removed.

•When an OST is removed from the file system, it will no longer be seen in Chroma Manager. Before removing the OST, manually remove all data from the OST. When an OST is removed, files stored on the OST will no longer be accessible.

4.Click Stop LNet to shut down the LNet networking layer and stop any targets running on this server.

5.Click Unload LNet to stop LNet, if it is running, and unload the LNet kernel module to ensure that it will be reloaded before any targets are started again.

(Clicking Start LNet will reload the LNet kernel module and start the LNet networking layer again.)

6.Click Force Remove to remove the record for the server in Chroma Manager without attempting to contact the server. Any targets that depend on this server will also be removed without any attempt to unconfigure them. This action should only be used if the server is permanently unavailable.

To manage Chroma Storage from the command line, you must run Chroma CLI commands on the Chroma Manager server. Do not run Lustre commands from the UNIX command line of a metadata or object storage server in a Chroma Storage appliance that is being managed using Chroma Manager.

Lustre commands can be used to manage metadata or object storage servers in an existing Lustre storage system that has been set up outside Chroma Manager and is being monitored, but not managed, in Chroma Manager.
