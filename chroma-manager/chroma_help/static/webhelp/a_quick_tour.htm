A Quick Tour

  

The Chroma Manager user interface is organized as a set of pages offering monitoring and management functionality:

Dashboard - Used for monitoring file systems at the file system level and server level to gather information for monitoring performance and other parameters.

Configuration - Contains a series of pages used to create, configure, and manage file systems:

•File Systems - Provides access to a step-by-step procedure to create a file system and features to configure the file system and file system components. Tasks such as starting, stopping, or removing an entire file system or starting, stopping, or removing a management, metadata, or object storage target in a file system can also be performed on this page.

•MGTs - Provides features to create and configure a management server.

•Volumes - Provides features to configure primary and failover servers in file systems with servers configured for high availability.

Volumes (also called LUNs or block devices) are the underlying units of storage used to create Lustre file systems.  Each Lustre target corresponds to a single volume. If servers in the volume have been configured for high availability, primary and secondary servers can be designated for a Lustre target. Only volumes that are not already in use as Lustre targets or local file systems are shown. A volume may be accessible on one or more servers via different device nodes, and it may be accessible via multiple device nodes on the same host.

•Servers - Allows a server to be added to Chroma Manager, provides server status information, and allows a server to be started, stopped, or removed.

•Storage - Provides a means to configure and view a custom storage system appliance provided by a storage solution provider. The features on this page are specific to the appliance provided by the storage solution provider.

Alerts – Shows active alerts and alert history.

History – Displays a history of events and allows filtering of events by host, severity, and event type.

Logs – Displays log information and allows filtering by host, date range, and messages from Lustre or all sources.
