Diagnostics Tool


Contents

Intel® Manager for Lustre* software includes a diagnostics tool that captures an extensive set of diagnostic information including all log files generated by the manager software and by Intel® Enterprise Edition Lustre*. This tool can be run on the manager server and on the managed, target servers.  The script must be run on each server node that you want to collect information for.  Communication between servers is not required.   The following information is collected:

•device detection info from "chroma-agent device_plugin --plugin=linux" (stdout and stderr separately)

•a dump of the pacemaker configuration

•all chroma logs

•an "rpm -qa"

•a "rabbitmqctl list_queues -p chromavhost"

•a "chroma-config validate"

•all a chroma_core_* DB tables except for stats and logs.

•all API resources (like spider_api in the API unit tests, but limit the length of the logs output).

How to run the diagnostics tool

This tool is already downloaded to each server running Intel® Manager for Lustre* software or present with the file system created running the manager software. To run the diagnostics tool:

1.Using SSH, log in to the server of interest.

2.Enter the command: $ chroma-diagnostics [ -v | --verbose ]

 

The tool places the diagnostic information it the following location and file:

/var/log/diagnostics_<time>.tar.gz

 

Send this file to your Intel Customer Support representative for further assistance.
