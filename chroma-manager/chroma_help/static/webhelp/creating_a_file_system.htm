Creating a Lustre* File System


Contents

Creating a new Lustre* file system, to be monitored and managed from the Intel® Manager for Lustre* software, involves the following four short procedures.

Note: All references herein to the manager refer to the Intel® Manager for Lustre* software.

1.Identify the storage servers to the manager.

2.Configure the volumes.

3.Configure the management target (MGT) (optional).

4.Create the Lustre file system.

5.View the file system.

Each procedure is detailed next. Follow these steps.

A. Identify the storage servers to the manager

Note that all authentication credentials are sent to the manager server via SSL and not saved to any disk.

To configure the servers to be used for the file system:

1.Go to Configuration > Servers  to display the Server Configuration page.

2.Click +Add server. The Add server dialog is displayed.

3.In the Hostname field, enter the name of the server to be added. Then click Continue. /// dan cannot proof beyond this point, SW hangs/reloads.///

4.In the Add Host dialog, you can use one of these three methods:

a.Click Existing Key to use an existing SSH private key present on this management server.  There must be a corresponding SSH public key on server you are adding.

b.Click Root Password and enter a root password for the server you are adding.  ///(is this established already?)

c.Click Another Key and enter a private key that corresponds to a public key present on the server you are adding.  If the key is encrypted, enter the passphrase needed to decrypt the private key.

5.Click Continue.

The manager checks that each server is up and connected to the network. It then does an audit of the storage resources on each server.  /// this sentence is vague.. ///  The manager then checks the networking, loads Lustre modules, and makes sure the Lustre networking layer is functioning. When all checks are completed, the server is fully qualified as a Lustre server.

6.To add another server, click Add Another and repeat steps 3-5.

7.Click Close.
B. Configure the volumes

To view the volumes that were discovered and make adjustments to default configurations, complete these steps:

1.Go to Configuration > Volumes to display the Volume Configuration page. A list of available volumes is displayed (if a volume does not contain unused block devices, it will not appear on this list).

2.For volumes that are configured for high availability (status is green), modify the default failover settings as needed by adjusting the options in the Primary Server and Failover Server columns.
C. (Optional) Configure the management target (MGT)

The management target can be configured using this procedure, or it can be configured as a step in the procedure for creating a new file system (see "D. Create the file system" below)

To configure the management target, complete these steps:

1.Go to Configuration > MGTs to display the MGT Configuration page.

2.Under New MGT, click Select storage and select the server for the MGT.

  Note: The MGT and metadata target (MDT) can be located on the same server. However, they cannot be located on the same volume on a server.

3.Click +Create new MGT to create the new MGT.

D. Create the file system

To create the file system, complete these steps:

1.Go to Configuration > File Systems to display the File System Configuration page.

2.Click New File System to display the New File System Configuration page.

3.In the File System Name field, enter the name of the new file system.

4.Choose a management target (MGT) from the table displayed.

If a management target has been created previously, the following options will be available. Use one of these options to select the MGT.

•If the MGT is to be installed on an existing server in the file system, use the drop down list provided to select the server to be used.

•If a new MGT is to be created, click Select Storage to display a list of available servers and then click the server to be used.

Note: The MGT and MDT can be located on the same server. However, they cannot be located on the same volume on a server.

5.Choose a metadata target (MDT) by clicking Select Storage and selecting the server to be used.

6.Choose the object storage targets (OSTs) for the file system by checking the boxes next to the targets to be included in the system.

7.Click Create file system now to create the file system.

8.To follow the process as the file system is created, click on Notifications on the left side of the screen and select Commands at the bottom of the notifications list.

Note: Intel® Manager for Lustre* software will automatically assign OST indices in a distributed fashion across servers to permit striping.  See OST Index Assignments Facilitate Striping.
E. View the new file system

1.To view the file system configuration, go to the Configuration page and select the name of the file system in the table displayed.

2.To view the dashboard metrics for the file system, go to Dashboard and select File Systems and the name of the file system in the fields displayed at the top of the page.

Note: For a new file system, some of the dashboard charts may appear blank until the file system has been running long enough to collect performance data.
